Variable List

Here we will define all of our 18 variables provided by our client.



Variable Definitions


Target Variable

  • Revenue: this variable is our target variable. This is what our model is predicting,if the shopping experience is revenue generating or not.

Numerical Variables

  • Administrative: This is the count of pages open in a given session with the tag "Administrative" (E-mail,banking,admin pages)
  • Administrative Duration: This is the total amount of time in seconds that a user spent of Administrative websites.
  • Informational: This is the count of pages open in a given session with the tag "Informational"
  • Informational Duration This is the total amount of time in seconds that a user spent of Informational websites.
  • Product Related: This is the count of pages open in a given session related to a product being sold "Product" (having another tab open for shoes).
  • Product Related Duration: This is the total amount of time in seconds that a user spent of Product Related websites.
  • Bounce Rates: The value of "Bounce Rate" feature for a web page refers to the percentage of visitors who enter the site from that page and then leave ("bounce") without triggering any other requests to the analytics server during that session.
  • Exit Rates: The value of "Exit Rate" feature for a specific web page is calculated as for all pageviews to the page, the percentage that were the last in the session.
  • Page Values: The "Page Value" feature represents the average value for a web page that a user visited before completing an e-commerce transaction.For example each time a user gets to the checkout form, they will increase their page value by 'x' amount.
  • Special Day: The "Special Day" feature indicates the closeness of the site visiting time to a specific special day (e.g. Mother’s Day, Valentine's Day) in which the sessions are more likely to be finalized with transaction. The value of this attribute is determined by considering the dynamics of e-commerce such as the duration between the order date and delivery date. For example, for Valentine’s day, this value takes a nonzero value between February 2 and February 12, zero before and after this date unless it is close to another special day, and its maximum value of 1 on February 8.

Categorical Variables

  • Month: Categorical variable for which month the current user visited your website (January and April Missing).
  • Operating Systems: Unlabeled categorical variable that distinguishes between operation systems e.g.(Mac,Windows,Linux).
  • Browser: Unlabeled categorical variable for browser type which tracks what browser a given visit was made with e.g.(Safari,Chrome,Firefox).
  • Region: Unlabeled categorical variable which tracks which region of the world a user is browsing from.
  • Traffic Type: Unlabeled categorical variable that assigns categories depending on traffic type.
  • Visitor Type: Type of visitor falls under three categories, Returning, New, other. We suspect that the other category refers to transactions and visits that were done from a third party platform and thus cannot be correctly classified.
  • Weekend :A binary variable to indicate whether the visit was made on a weekend. Will take the value of True if it was made on a weekend and False if it was during a weekday.
